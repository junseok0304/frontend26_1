1. 약타입 언어와 강타입 언어. 
js는 약타입 언어이다. 
js에서는 3 과 "3"이 같은지 비교하면 같다고 나온다.
a == b 의 결과는 true 이다.
하지만 a === b 의 결과는 false 이다.
js에서는 변수의 타입이 동적으로 결정되기 때문에, 변수에 어떤 타입의 값이든 할당할 수 있다. 
a != b 의 결과는 false 이다.

2. === 연산자는 == 연산자와 기능이 같고 !== 연산자는 != 연산자와 기능이 같다. 
== 나 != 로 비교할 때는 타입을 엄격하게 비교하지 않음 
=== 나 !== 로 비교할 때는 타입을 엄격하게 비교함

3. typeof 연산자를 이용해 값의 타입을 확인할 수 있다. 
값의 타입을 문자열로 알려준다. 
console.log(typeof 3); // number
console.log(typeof "3"); // string
console.log(typeof true); // boolean
약타입 언어인 js에서는 어떤 변수에 number, string, boolean 값을 번갈아 대입해도 된다.

4. undefined : js에 존재하는 키워드로 아직 값이 대입되지 않은 변수의 값이다. 
let a;
console.log(a); // undefined, 값도 타입도 undefined 이다.

5. null : js에 존재하는 키워드로 변수에 null 값을 대입할 수 있다. 
값이 있고, 그 값이 null이다 라는 뜻. 변수 값이 null 인지 비교할 수 있다. 

let a = null;
console.log(a); // null, 값은 null 이지만 타입은 object 이다.

undefined vs null : 둘 다 변수 값이 없음을 의미하나, 
null은 값이 존재하지만 그 값이 null 이라는 뜻이고, 
undefined는 값도 타입도 undefined 라는 뜻이다.

6. 기본 자료형 (primitive type) : number, string, boolean, undefined, null
값 타입(value type) 이라고도 한다. 
js에서는 정수형과 실수형을 구분하지 않고 모두 number 라는 타입으로 표현한다.
undefined도 자료형이다.

7. 복합 자료형 (complex type) : function, object(js 배열은 object 타입의 객체이다.)
참조형(refrence type) 이라고도 한다. 

js 에서는 function 도 자료형이다.
js에서는 함수가 값이라는 말이다. 함수를 변수에 대입할 수 있고 파라미터로 함수를 전달할 수도 있다.
어떤 변수 값의 자료형이 함수 일 수 있다.

8. boolean 자료형
true false 값으로 나뉘며 typeof 연산자로 확인하면 boolean 이라고 나온다.

9. false에 해당하는 값 
js에서는 boolean 타입이 아닌 표현식도 if 문이나 while 문의 조건식에 사용될 수 있다. 

조건식에 사용된 값이 불린 타입이 아닌 경우, 
그 값이 NaN, 0, null, undefined, " " 이면 false 로 취급된다. 
그 외 나머지 값은 true 로 취습된다. 

NaN (Not a Number, 계산 결과가 숫자가 아닐 때 그 값은 NaN이다.)

조건식? 값1:값2
조건식이 true면 값1이 결과이고, 
조건식이 false면 값2가 결과이다.

참고) lazy evaluation 
&& || 연산자가 포함된 표현식에서, 실행할 필요가 없는 부분을 실행하지 않는 것을 lazy evaluation 이라고 한다. 
C, java, js 들 대부분의 언어는 lazy evaluation 한다. 

표현식1 || 표현식2
표현식 1의 값이 true 이면, 결과는 true이다. 그래서 표현식 2는 미실행, 위 OR식의 값은 표현식 1의 값이다. 
표현식 1의 값이 false이면 결과는 표현식 2의 값이다. 

## number 함수 
parseInt
parseFloat
Number

parseInt 함수는 소숫점 아래 값은 버린다. 
Number 함수 이름의 첫문자가 대문자임에 주의하자. 

let s = "123.456"; 
let a = parseInt(s); 
console.log(a);	// 123 

let b = parseFloat(s); 
console.log(b);	// 123.456 

let c = Number(s); 
console.log(c);	// 123.456 

## 형 변환 (number -> string)
toString
String

let a = 123.456; 
let s = a.toString(); 
console.log(s);		// "123.456" 
console.log(typeof s);	// "string" 

s = String(a); 
console.log(s); 		// "123.456" 
console.log(typeof s);	// "string" 

## NaN
js 에서 NaN 키워드는 계산 결과가 숫자가 아님을 의미한다. 

let a = 3 * "ABC"; 
console.log(a);		// NaN 
console.log(typeof a);	// "number" 

let b = Number("abc"); 
console.log(b);		// NaN 
console.log(typeof b);	// "number" 

NaN을 값으로 보려고 하면 NaN이지만 타입은 넘버임 console.log(typeof a); // "number"

유효숫자 자릿수 
숫자 타입 변수의 크기는 64비트이고, 실수형태로 저장되기에 유효숫자 자릿수 문제가 발생한다.
정수는 15자리 까지 정확하다.
15자리가 넘는 정수는 유효숫자 자릿수 부족 때문에 오차가 발생한다.
