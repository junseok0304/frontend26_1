1. 약타입 언어와 강타입 언어. 
js는 약타입 언어이다. 
js에서는 3 과 "3"이 같은지 비교하면 같다고 나온다.
a == b 의 결과는 true 이다.
하지만 a === b 의 결과는 false 이다.
js에서는 변수의 타입이 동적으로 결정되기 때문에, 변수에 어떤 타입의 값이든 할당할 수 있다. 
a != b 의 결과는 false 이다.

2. === 연산자는 == 연산자와 기능이 같고 !== 연산자는 != 연산자와 기능이 같다. 
== 나 != 로 비교할 때는 타입을 엄격하게 비교하지 않음 
=== 나 !== 로 비교할 때는 타입을 엄격하게 비교함

3. typeof 연산자를 이용해 값의 타입을 확인할 수 있다. 
값의 타입을 문자열로 알려준다. 
console.log(typeof 3); // number
console.log(typeof "3"); // string
console.log(typeof true); // boolean
약타입 언어인 js에서는 어떤 변수에 number, string, boolean 값을 번갈아 대입해도 된다.

4. undefined : js에 존재하는 키워드로 아직 값이 대입되지 않은 변수의 값이다. 
let a;
console.log(a); // undefined, 값도 타입도 undefined 이다.

5. null : js에 존재하는 키워드로 변수에 null 값을 대입할 수 있다. 
값이 있고, 그 값이 null이다 라는 뜻. 변수 값이 null 인지 비교할 수 있다. 

let a = null;
console.log(a); // null, 값은 null 이지만 타입은 object 이다.

undefined vs null : 둘 다 변수 값이 없음을 의미하나, 
null은 값이 존재하지만 그 값이 null 이라는 뜻이고, 
undefined는 값도 타입도 undefined 라는 뜻이다.

6. 기본 자료형 (primitive type) : number, string, boolean, undefined, null
값 타입(value type) 이라고도 한다. 
js에서는 정수형과 실수형을 구분하지 않고 모두 number 라는 타입으로 표현한다.
undefined도 자료형이다.

7. 복합 자료형 (complex type) : function, object(js 배열은 object 타입의 객체이다.)
참조형(refrence type) 이라고도 한다. 

js 에서는 function 도 자료형이다.
js에서는 함수가 값이라는 말이다. 함수를 변수에 대입할 수 있고 파라미터로 함수를 전달할 수도 있다.
어떤 변수 값의 자료형이 함수 일 수 있다.
